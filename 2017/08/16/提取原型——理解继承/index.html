javascript的继承

js没有”子类”和”父类”的概念，也没有”类”（class）和”实例”（instance）的区分，全靠一种很奇特的”原型链”（prototype chain）模式，来实现继承。

所有的实例对象共享同一个prototype属性，这个属性包含一个对象，那么从外界看起来，prototype属性的对象就好像是实例对象的原型，而实例对象则好像”继承”了prototype对象一样。

有趣的案例

首先我们构造一个煮咖啡对象，分为四个步骤：

  1. 把水煮沸
  2. 用沸水泡咖啡
  3. 把咖啡倒进杯子
  4. 加糖和牛奶

然后我们再构造一个泡茶对象，也分为四个步骤：

  1. 把水煮沸
  2. 用沸水浸泡茶叶
  3. 把茶倒进杯子
  4. 加柠檬

这时候我们会发现两个对象是有一些通用的部分的。那么我们接下去可以对公共部分进行抽象：

  1. 咖啡/茶 ——> 抽象成”饮料”(Beverage)
  2. 把水煮沸 ——> 完全一致(boilWater)
  3. 用沸水泡咖啡/用沸水浸泡茶叶 ——> 抽象成“泡”(brew)
  4. 把咖啡倒进杯子/把茶倒进杯子 ——> 抽象成“倒进杯子”(pourInCup)
  5. 加糖和牛奶/加柠檬 ——> 抽象成“加调料”(addCondiments)
